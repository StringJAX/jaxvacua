# Tutorial-notebook layout template

Every tutorial notebook under `documentation/source/notebooks/` ships with
this layout.  Copy this file as a starting point when authoring a new
notebook, and use the worked reference (`01_basics/02_jaxvacua_overview.ipynb`)
when in doubt about prose density or figure framing.

---

## File-naming convention

* **Filename**: `<NN>_<slug>.ipynb` where `<NN>` is the global notebook
  number (zero-padded two digits, sequential 01–16 across the four
  thematic subdirs) and `<slug>` is `lower_snake_case`.
* **Subdirs**: only the four thematic dirs are valid:
  - `01_basics/` (notebooks 01–04)
  - `02_vacuum_finding/` (notebooks 05–08)
  - `03_geometry_and_limits/` (notebooks 09–11)
  - `04_analysis_and_pipelines/` (notebooks 12–16)
* **Special**: `quickstart.ipynb` is the only unnumbered notebook (lives
  at `notebooks/quickstart.ipynb`).

---

## Required cell sequence

The following five sections are mandatory in every notebook.  Optional
sections (worked examples, appendices) sit between **§ Setup** and
**§ Take-aways**.

| #   | Cell type | Content                                                       |
|-----|-----------|---------------------------------------------------------------|
| 1   | markdown  | `# <Title>` — single H1 heading, no formulae, ≤ 60 chars       |
| 2   | markdown  | One-paragraph **abstract** (`> blockquote`, ≤ 4 sentences)    |
| 3   | markdown  | `## Outline` — numbered TOC linking to each H2 section         |
| 4   | markdown  | `## Setup` heading                                            |
| 5   | code      | imports + fixtures + (warnings filter / x64 toggle if needed) |
| 6+  | markdown + code | Body sections — `## <Section>` headings with prose-then-code rhythm |
| -2  | markdown  | `## Take-aways` — 3–5 bullets summarising what the reader retains |
| -1  | markdown  | (optional) `## Further reading` — links to follow-up notebooks |

## Style requirements

### Cells

* **No bare code cells at the top.** Every code block is preceded by a
  prose explanation answering "why this code, here".
* **One operation per code cell.** If a cell does two unrelated things
  (load a fixture *and* define a helper, say), split it.
* **Outputs.** Heavy outputs (large arrays, full plots) are stripped
  before commit via the `nbstripout` git filter (already installed —
  re-run `nbstripout --install` on a fresh clone if missing).  Commit
  final outputs *only* on the sign-off cycle for that notebook.

### Plots

* Use `notebooks/plot_utils.py` for shared styling.  Default DPI is **130**
  (publication-figure recipes in NB14 use **200–300**).
* Default colormap: `viridis`.  Default font family: `serif`.
* Figures rendering more than ~50 KB of pixel data should pass
  `rasterized=True` to keep PDF / HTML builds light.

### Math

* Inline math in `$...$`; display math in `$$...$$`.  No `\begin{align}`
  blocks unless multi-line equations are unavoidable (Sphinx + nbconvert
  render both, but `$$` is more portable).
* **First use** of every recurring symbol gets a short definition (see
  the planned cross-notebook glossary at `notebooks/_glossary.md`).
  Recurring symbols: `W₀`, `z_cf`, `g_s`, `N_flux`, ISD, AISD, PFV, AFV.

### Cross-references

* Internal anchors are auto-generated by Sphinx as
  `lower-hyphenated-section-title`.  Outline links must use those exact
  slugs.
* Inter-notebook links are relative paths from the current subdir, e.g.
  `[NB02 jaxvacua_overview](../01_basics/02_jaxvacua_overview.ipynb)`.

### Imports

* **Single import block** in `## Setup`.  Don't re-import in body cells
  unless a cell is meant to be self-contained for copy-paste demo.
* Standard preamble: `warnings.filterwarnings("ignore",
  DeprecationWarning)` + `jax.config.update("jax_enable_x64", True)` for
  any notebook that builds a JAXVacua model.

### Audience scoping

* **`01_basics/`** and `quickstart.ipynb` — assume zero JAXVacua
  knowledge; explain every API on first use.
* **`02_vacuum_finding/`**, **`03_geometry_and_limits/`** — assume the
  reader has worked through `01_basics/`; cross-link rather than re-explain.
* **`04_analysis_and_pipelines/`** — assume working knowledge of the upstream
  notebooks; focus on the analysis / pipeline workflow rather than API
  basics.

## Sign-off checklist (per-notebook)

Before requesting sign-off on a notebook, verify:

- [ ] Title (H1), abstract, outline, setup, body, take-aways all present.
- [ ] No bare code cells at the top; every code cell preceded by prose.
- [ ] All cross-references resolve (inter-notebook + outline anchors).
- [ ] `jupyter nbconvert --execute --to notebook --output-dir=/tmp/nbtest
  documentation/source/notebooks/<path>.ipynb` runs clean (no errors).
- [ ] `make -C documentation html SPHINXOPTS="-W -q"` succeeds (warnings
  treated as errors).
- [ ] Cell outputs committed (not stripped) so the rendered docs show
  results.  `nbstripout` re-strips them on the next commit if the source
  hasn't changed — that's expected.
